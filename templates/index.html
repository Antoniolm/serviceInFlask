Patch principales de la api

  • /users/id_user
    • GET -> Sin ID: obtiene todos los usuarios del sistema Con ID: obtiene el usuario con la ID especificada
    • POST -> Sin ID: crea un nuevo usuario en el sistema Con ID: actualiza la información de un usuario
    • DELETE -> elimina el usuario con la ID especificada ( metodo activo solo cuando se especifica una ID)
  • /products/id_product
    • GET -> Sin ID: obtiene todos los productos del sistema Con ID: obtiene el producto con la ID especificada
    • POST -> Sin ID: crea un nuevo producto en el sistema Con ID: actualiza la información de un producto
    • DELETE -> elimina el producto con la ID especificada ( metodo activo solo cuando se especifica una ID)
  • /pharmacies/id_pharmacy
    • GET -> Sin ID: obtiene todos los farmacias del sistema Con ID: obtiene la farmacia con la ID especificada
    • POST -> Sin ID: crea una nueva farmacia en el sistema Con ID: actualiza la información de una farmacia
    • DELETE -> elimina la farmacia con la ID especificada ( metodo activo solo cuando se especifica una ID)
  • /order/id_order
    • GET -> Sin ID: obtiene todos las peticiones del sistema
    • POST -> Sin ID: crea una nueva petición en el sistema
    • DELETE -> elimina la farmacia con la ID especificada ( metodo activo solo cuando se especifica una ID)
  • /login
    • POST -> Permite comprobar si un conjunto (usuario,contraseña) existe en la base de datos
  • /requestProduct
    • POST -> Permite actualizar los productos tras un pedido
